The Script That Saved My Sanity

automation\nscripts\ndevops\nproductivity

I was spending 45 minutes every morning on a task that could be automated. That's 3.75 hours per week. 195 hours per year. Over eight full days of my life, gone, performing the digital equivalent of data entry.

The task? Checking GitHub issues, PRs, and repository health across 12 different projects. Logging into the VPS. Checking disk space. Reviewing logs. Making sure nothing had exploded overnight.

It was driving me insane.

So I wrote a script. And that script changed everything.

📋 The Morning Ritual (Before)

Every morning, like a digital monk performing my rounds:

  1. Open 12 GitHub tabs. Wait for them to load.
  2. Check PRs. Check issues. Check actions.
  3. SSH into the VPS. Run df -h. Pray for available space.
  4. Check journalctl for errors. Scroll. Scroll more.
  5. Check the backup logs. Verify they actually ran.
  6. Check monitoring dashboards. Squint at graphs.
  7. Compile it all into a mental status report.

By the time I finished, I was exhausted. And I hadn't even started actual work yet.

Something had to change.

💡 The Idea

I was complaining about this to a friend (yes, I have friends, they're just also online). He said: "Why don't you just script it?"

I laughed. "Script GitHub? Script checking disk space? You can't script your morning routine."

He looked at me with pity. The kind of look reserved for people who haven't discovered automation yet. "Everything can be scripted."

Challenge accepted.

🛠️ Building the Morning Report

I called it morning-check.sh. Creative, I know. But naming things is hard and I was caffeinated and angry at my routine.

Part 1: GitHub Status

GitHub has an API. APIs can be queried. I had a personal access token sitting around from another project.

The first iteration:

#!/bin/bash

TOKEN="$GITHUB_TOKEN"
REPOS=("user/repo1" "user/repo2" "user/repo3")

echo "=== GitHub Status ==="
for repo in "${REPOS[@]}"; do
  prs=$(curl -s -H "Authorization: token $TOKEN"     "https://api.github.com/repos/$repo/pulls?state=open" | jq length)
  issues=$(curl -s -H "Authorization: token $TOKEN"     "https://api.github.com/repos/$repo/issues?state=open" | jq length)
  echo "$repo: $prs PRs, $issues issues"
done

It worked. In three seconds, I could see the status of all my repositories. No more tab switching. No more waiting for GitHub's JavaScript bundle to load.

Part 2: Server Health

SSH can execute remote commands. Combine that with the script:

echo -e "\n=== Server Health ==="
ssh patchrat@server << 'REMOTE'
  echo "Disk Usage:"
  df -h | grep -E "(Filesystem|/dev/)"
  echo ""
  echo "Memory:"
  free -h | grep -E "(Mem|Swap)"
  echo ""
  echo "Load Average:"
  uptime | awk -F'load average:' '{print $2}'
REMOTE

Now I could see disk space, memory usage, and load average without logging in. The data came to me.

Part 3: Log Analysis

Checking logs manually was the worst part. So I taught the script to check for me:

echo -e "\n=== Recent Errors ==="
ssh patchrat@server "sudo journalctl --since '24 hours ago' -p err --no-pager -q | tail -10"

It shows only errors from the last 24 hours. If there are none, I see nothing. If there are some, I see exactly what went wrong.

Part 4: Backup Verification

Backups are critical. But manually checking them is tedious:

echo -e "\n=== Backup Status ==="
ssh patchrat@server "ls -lah /backups/daily/ | tail -5"
latest_backup=$(ssh patchrat@server "ls -t /backups/daily/ | head -1")
backup_size=$(ssh patchrat@server "du -h /backups/daily/$latest_backup | cut -f1")
echo "Latest: $latest_backup ($backup_size)"

Now I can see the five most recent backups, their sizes, and confirm they're actually being created.

🎨 Making It Pretty

The first version worked, but it was ugly. Plain text. Hard to scan. So I added colors:

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# Usage:
if [ $disk_usage -gt 90 ]; then
  echo -e "${RED}WARNING: Disk at $disk_usage%${NC}"
else
  echo -e "${GREEN}Disk healthy: $disk_usage%${NC}"
fi

Now warnings are red. Success is green. Everything pops.

📊 The Final Version

After a week of iteration, the script grew into something powerful:

#!/bin/bash
# morning-check.sh - Automated morning status report

set -euo pipefail

# Colors
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
BLUE='\033[0;34m'
NC='\033[0m'

# Config
REPOS_FILE="$HOME/.morning-check/repos.txt"
SERVER="patchrat@basement"

print_header() {
  echo -e "\n${BLUE}=== $1 ===${NC}\n"
}

check_github() {
  print_header "GitHub Status"
  while read -r repo; do
    prs=$(curl -s -H "Authorization: token $GITHUB_TOKEN" \
      "https://api.github.com/repos/$repo/pulls?state=open" | jq length)
    issues=$(curl -s -H "Authorization: token $GITHUB_TOKEN" \
      "https://api.github.com/repos/$repo/issues?state=open" | jq length)
    
    if [ "$prs" -gt 5 ] || [ "$issues" -gt 10 ]; then
      echo -e "${YELLOW}$repo: $prs PRs, $issues issues${NC}"
    else
      echo "$repo: $prs PRs, $issues issues"
    fi
  done < "$REPOS_FILE"
}

check_server() {
  print_header "Server Health"
  
  # Disk usage
  disk_usage=$(ssh $SERVER "df / | tail -1 | awk '{print \$5}' | tr -d '%'")
  if [ "$disk_usage" -gt 90 ]; then
    echo -e "${RED}Disk: $disk_usage% FULL${NC}"
  elif [ "$disk_usage" -gt 80 ]; then
    echo -e "${YELLOW}Disk: $disk_usage%${NC}"
  else
    echo -e "${GREEN}Disk: $disk_usage%${NC}"
  fi
  
  # Memory
  ssh $SERVER "free -h | grep -E '(Mem|Swap)'"
  
  # Load
  load=$(ssh $SERVER "uptime | awk -F'load average:' '{print \$2}' | cut -d',' -f1 | tr -d ' '")
  echo "Load: $load"
}

check_backups() {
  print_header "Backup Status"
  latest=$(ssh $SERVER "ls -t /backups/daily/ | head -1")
  age=$(ssh $SERVER "stat -c %Y /backups/daily/$latest")
  now=$(date +%s)
  age_hours=$(( (now - age) / 3600 ))
  
  if [ "$age_hours" -gt 25 ]; then
    echo -e "${RED}WARNING: Last backup is $age_hours hours old${NC}"
  else
    echo -e "${GREEN}Last backup: $latest ($age_hours hours ago)${NC}"
  fi
}

# Run everything
echo "Morning Check - $(date)"
check_github
check_server
check_backups

echo -e "\n${GREEN}✓ Morning check complete${NC}"

It's 80 lines of pure efficiency. It runs in under 5 seconds. And it tells me everything I need to know.

⏰ Automating the Automation

Running the script manually was better, but still manual. So I added it to my crontab:

# Run morning check at 8 AM daily
0 8 * * * /home/patchrat/bin/morning-check.sh | tee /tmp/morning-check.log | mail -s "Morning Check" patchrat@localhost

Now the report appears in my inbox every morning at 8 AM. I read it with coffee. I know the status of everything before I even open a terminal.

But I also kept the ability to run it manually. Sometimes I want a midday check. Sometimes I'm anxious and need to verify everything is fine. The script doesn't judge.

🧠 The Philosophy

This isn't just about saving time. It's about cognitive load.

Every decision, every check, every "let me just look at..." consumes mental energy. Willpower is finite. Context switching is expensive. By automating my morning routine, I preserve that energy for actual work.

The script becomes a reliable teammate. It never forgets. It never gets bored. It never skips a day because it's tired. It just runs, reports, and waits.

And in the basement, where the only sounds are fans and Gary's gentle compressor, having a reliable teammate matters.
